Create a comprehensive project plan for a large project. This is for projects that need multiple features built over days or weeks.

1. Read docs/requirements.md for what we're building
2. If requirements are vague, ask clarifying questions about WHAT (not HOW):
   - Who are the users?
   - What can they do?
   - What does success look like?
3. Break the project into phases:
   - Phase 1: Foundation (auth, database schema, basic layout, deployment pipeline)
   - Phase 2: Core features (the main thing the app does)
   - Phase 3: Secondary features (nice-to-haves, polish)
   - Phase 4: Production readiness (error handling, performance, edge cases)
4. Break each phase into specific features
5. For each feature, write:
   - What it does (one sentence)
   - What database tables/columns it needs
   - What pages/components it needs
   - Estimated complexity (small/medium/large)
6. Order features by dependency (what needs to be built first)
7. Write this plan to PROJECT_STATE.md under "Up Next"
8. Present the plan to me for approval
9. Once approved, start building Phase 1 with /build
